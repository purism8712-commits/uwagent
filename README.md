# 신계약 인수기준 반영 Agent

보험 신계약 가이드라인 엑셀을 업로드하면, 공통 코어가 원문을 읽어 표준 템플릿으로 정리하고 변경 내용을 반영한 초안을 만들어 주는 Next.js 앱입니다.

이 프로젝트의 목적은 `기획 / 지원 / 심사` 에이전트가 같은 기준을 같은 구조로 해석할 수 있도록, 원본 자료를 먼저 구조화된 입력으로 정리하는 것입니다.

## 핵심 역할

- 가이드라인 엑셀 여러 개를 업로드해 `전체 통합 마스터 파일` 생성
- 변경 내용을 자연어 / 표 / 파일로 입력
- 보험코드, 특약명, 주석, 예외조건을 기준으로 변경 대상 확인
- 핵심 확인 질문과 검토 메모를 통해 변경 여부를 다시 확인
- 최종 초안과 다운로드 파일 생성

## 화면 흐름

1. **STEP 0**: 가이드라인 엑셀 업로드 및 전체 통합 마스터 파일 생성
2. **STEP 1**: 업로드한 파일의 표준화 결과 미리보기
3. **STEP 2**: 변경 내용 입력
4. **STEP 3**: 핵심 확인 질문 및 검토 메모 확인
5. **STEP 4**: 초안 생성 결과 확인
6. **STEP 5**: 통합 마스터 / 상품 추출 다운로드

## 주요 기능

- 로그인 화면과 부서별 접근 제어
- 상품명 필터, 특약명 필터
- 주석 ID 툴팁 확인
- 통합 마스터 미리보기 탭 전환
- 변경 대상 특약만 반영하는 초안 생성
- 공통 주석을 Note Master / Rule-Note Map 으로 분리

## 로컬 실행

```bash

```


## 테스트

```bash
npm test
```

## 환경 변수

이 앱은 BizRouter 연동을 위한 환경 변수를 사용합니다. 실제 값은 `.env` 또는 실행 환경에 설정하고, 저장소에는 커밋하지 않습니다.

- `BIZROUTER_API_KEY`
- `BIZROUTER_BASE_URL`
- `BIZROUTER_MODEL`

## 프로젝트 구조

- `app/` - Next.js 라우트 및 API
- `components/` - 화면 컴포넌트
- `lib/` - 파싱, 초안 생성, 주석/변경 로직
- `tests/` - 주요 동작 검증 테스트
- `data/` - 업로드 파일 및 출력 산출물

## 작업 원칙

- 원문에 없는 값은 임의로 확정하지 않습니다.
- 애매한 변경은 `검토 필요`로 남깁니다.
- 주석과 규칙은 분리하고, 연결 관계를 따로 관리합니다.
- 최종 승인 전까지는 초안 상태를 유지합니다.

## 참고

이 프로젝트는 공통 코어 전처리 역할에 집중합니다.  
기획 / 지원 / 심사 에이전트는 이후 단계에서 같은 구조의 마스터 데이터를 참조해 동작합니다.
