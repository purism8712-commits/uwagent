# 공통 코어 앱 초안확정 후 엑셀 출력/요약 설계서

## 목적

현재 2단계 검토 화면의 `초안확정` 버튼을 실제 동작으로 확장해, 사용자가 확인 질문에 답변한 뒤 공통 코어 양식 엑셀을 다운로드하고 화면에서 변경 최종 요약도 함께 확인할 수 있게 한다.

## 범위

- `초안확정` 클릭 후 서버 측에서 공통 코어 초안 데이터 생성
- `Rule Master`, `Note Master`, `Rule-Note Map`, `Change Log` 시트를 가진 엑셀 파일 생성
- 화면 내 `변경 최종 요약` 섹션 표시
- 엑셀 다운로드와 화면 요약은 분리 제공

## 사용자 흐름

1. 사용자가 입력 화면에서 엑셀 또는 자연어를 입력한다.
2. `입력완료`를 눌러 검토 화면으로 이동한다.
3. 검토메모를 확인하고 질문에 답한다.
4. `초안확정` 버튼을 누른다.
5. 앱은 서버에 현재 입력과 답변을 보낸다.
6. 서버는 공통 코어 양식 초안 데이터를 만든다.
7. 화면에는 `변경 최종 요약`이 보인다.
8. 사용자는 별도 `엑셀 다운로드` 버튼으로 초안 파일을 내려받는다.

## 출력 형태

### 1. 엑셀 다운로드

- 시트 4개 고정
  - `Rule Master`
  - `Note Master`
  - `Rule-Note Map`
  - `Change Log`
- 현재 단계에서는 샘플 데이터 기반 생성
- 이후 BizRouter 실제 응답으로 대체 가능하게 데이터 생성 함수 분리

### 2. 변경 최종 요약

화면 내 카드 또는 섹션으로 별도 표시

- 변경 대상
- 변경 전 값
- 변경 후 값
- 검토 답변 반영 내용
- 남은 검토 필요 항목

## 구현 방향

- Next.js App Router 유지
- Route Handler 추가
- 클라이언트는 `초안확정` 클릭 시 POST 요청
- 서버는 엑셀 파일 바이트와 요약 JSON을 함께 만들 수 있는 구조 사용
- 클라이언트는 먼저 요약 상태를 갱신하고, 다운로드 버튼으로 별도 파일 요청

## API 초안

### `POST /api/draft-summary`

입력:

- 업로드 파일명
- 자연어 입력값
- 질문 답변

출력:

- 요약 데이터
- 다운로드 준비 상태

### `POST /api/draft-export`

입력:

- 업로드 파일명
- 자연어 입력값
- 질문 답변

출력:

- 공통 코어 양식 `.xlsx` 파일

## 화면 변화

### 검토 화면

- `초안확정` 버튼 클릭 전:
  - 질문 입력 가능
- `초안확정` 버튼 클릭 후:
  - `변경 최종 요약` 섹션 노출
  - `엑셀 다운로드` 버튼 노출 또는 활성화
  - 필요 시 처리 중 상태 표시

## 상태 모델 추가

- `isSubmitting`
- `isDraftReady`
- `finalSummary`

## 테스트 기준

- `초안확정` 클릭 시 요약이 렌더링된다
- 요약에는 변경 전/후와 반영 답변이 표시된다
- `엑셀 다운로드` 버튼 클릭 시 `.xlsx` 파일 응답이 내려온다
- 빌드와 기본 상호작용이 유지된다
