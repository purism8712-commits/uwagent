# STEP Flow Layout Design

## Goal
공통 코어 화면을 `STEP 1`부터 시작하는 세로형 단일 흐름으로 재배치한다.
사용자는 위에서 아래로 `STEP 1 → STEP 2 → STEP 3 → STEP 4 → STEP 5` 순서로 진행하며,
`초안생성`을 눌렀을 때만 `STEP 4`와 `STEP 5` 카드가 아래로 펼쳐진다.

## User Flow
1. `STEP 1. 전체 통합 마스터 파일 만들기` 카드에서 가이드라인 엑셀 업로드와 미리보기를 수행한다.
2. `STEP 2. 변경내용 입력` 카드에서 변경된 엑셀, 표, 자연어를 입력한다.
3. `입력완료`를 누르면 아래에 `STEP 3. 검토 메모 및 확인질문` 단일 카드가 펼쳐진다.
4. `STEP 3`에서 질문 답변을 모두 채운 뒤 `초안생성`을 누르면 아래에 `STEP 4` 카드가 펼쳐진다.
5. `STEP 4` 아래에 `STEP 5. 통합 마스터 다운로드 / 상품 추출 다운로드` 카드가 이어서 나타난다.

## Layout Rules
- 화면은 세로 단일 흐름을 기본으로 한다.
- `STEP 1`은 업로드, 생성, 미리보기 기능을 한 카드 안에 유지한다.
- `STEP 2`는 변경 입력 전용 카드로 분리한다.
- `STEP 3`는 질문 리스트와 검토 메모를 한 카드에 합치고, 내부에서 스크롤되도록 한다.
- `STEP 4`는 `초안생성` 이후 생성 결과와 다음 단계 안내를 담는 확장 카드로 둔다.
- `STEP 5`는 다운로드 행동을 모은 마지막 카드로 두고, `통합 마스터 다운로드`와 `상품 추출 다운로드`를 함께 배치한다.

## Interaction Rules
- `입력완료` 전에는 `STEP 3` 이후 카드가 보이지 않거나 비활성 상태여야 한다.
- `초안생성` 전에는 `STEP 4`와 `STEP 5`가 표시되지 않거나, 표시되더라도 비활성 상태여야 한다.
- 다운로드 버튼은 `STEP 5`에서만 노출한다.
- 진행 중인 단계는 상단의 진행 표시와 본문 카드 순서가 서로 일치해야 한다.

## Content Rules
- `STEP 3`는 검토 메모와 확인 질문을 한 카드에 묶어 보여준다.
- 질문은 카드 내부에서 스크롤 가능해야 하며, 질문 수가 많아져도 카드 높이가 과도하게 늘어나지 않도록 한다.
- `STEP 4`는 초안 생성 결과와 다음 행동을 요약한다.
- `STEP 5`는 결과 다운로드 행동만 강조하고, 설명은 최소화한다.

## Success Criteria
- 사용자가 흐름을 한 번에 이해할 수 있다.
- `입력완료`와 `초안생성`이 각각 다음 카드의 출현 기준이 된다.
- 질문과 다운로드가 각 단계에 깔끔하게 분리된다.
- 위에서 아래로 읽는 순서가 실제 업무 진행 순서와 일치한다.
