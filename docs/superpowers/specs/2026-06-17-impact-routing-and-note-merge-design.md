# 영향도 판별 및 주석 병합 설계서

## 목적

변경 입력이 들어왔을 때, 전체 통합 마스터 파일을 기준으로 영향 받는 특약과 주석을 먼저 판별한다.  
`소액암진단 단일건 1000 → 2000` 같은 한도 변경은 해당 특약에 우선 반영하고,  
연결된 주석이 같이 바뀔 수 있으면 즉시 반영한다.  
주석 연결이 모호하거나 여러 특약에 걸치면 사람 확인 질문으로 넘긴다.

## 배경

현재 공통 코어는 업로드된 가이드라인 엑셀을 통합 마스터로 정리하고, 변경 입력을 받아 초안을 만드는 역할을 한다.  
다만 변경내용이 들어오면 한 특약만 바꿔야 하는 경우에도 연관 특약 전체에 퍼지는 문제가 있었다.  
또한 표 안의 `비고`와 표 아래의 `주1)` 류 본문 주석이 혼용되어 있어, 주석 원문을 실제 설명문으로 복원할 필요가 있다.

## 목표

1. 변경 입력에서 `한도 변경`, `주석 변경`, `예외 변경`을 분리한다.
2. 전체 통합 마스터에서 영향을 받는 특약만 좁힌다.
3. 즉시 반영 가능한 항목은 초안에 자동 반영하고 `변경내용`에 기록한다.
4. 연결이 불명확한 주석이나 여러 특약에 걸친 항목은 확인 질문으로 분기한다.
5. 표 안의 `비고`와 표 아래의 `주1)` 본문 주석을 함께 `Note Master`에 합친다.

## 범위

### 포함

- 변경 텍스트/표/업로드 파일에서 영향 특약 후보 판별
- 특약별 한도 변경 반영
- 특약별 주석 텍스트 병합
- 확인 질문 생성
- `변경내용` 항목 기록
- `Note Master`에 비고 + 본문 주석 병합

### 제외

- 인수기준 최종 승인
- 운영 반영 자동 실행
- 외부 시스템 입력
- 실시간 협업 편집

## 설계 원칙

- 먼저 후보를 좁히고, 그다음에 LLM으로 최종 판단한다.
- 특약명과 보험코드가 명확하면 그 특약만 변경한다.
- 주석은 해당 특약에 연결되어 있을 때만 자동 병합한다.
- 연결이 불확실하면 반영하지 않고 질문으로 남긴다.
- `비고`는 짧은 참조값이고, 본문 주석은 설명문이므로 둘을 분리했다가 다시 합친다.

## 변경 입력 판별

입력은 다음 세 갈래로 분리한다.

1. `한도 변경`
   - `1000 → 2000`, `상향`, `인별합산 조정` 같은 표현
   - 대상 특약을 정확히 찾으면 즉시 Rule Master의 해당 행만 수정한다.

2. `주석 변경`
   - `주1`, `주2,3`, `66세 이상 예외`처럼 주석 라벨 또는 조건이 들어간 표현
   - 표 아래 본문 주석을 찾아 텍스트를 확장한다.

3. `혼합 변경`
   - 한도와 주석이 함께 바뀌는 경우
   - 먼저 한도 반영 여부를 판단하고, 연결된 주석도 자동 반영 가능하면 같이 반영한다.

## 영향도 판별 로직

### 1차 규칙 필터

- 변경문장에 `소액암진단`, `주보험`, `파워수술 1종` 같은 특약명이 있으면 해당 특약을 우선 후보로 잡는다.
- 보험코드가 있으면 특약명보다 우선한다.
- `주1)`, `주2,3)` 같은 라벨은 `Note Master` 후보로 분리한다.

### 2차 LLM 판별

- LLM은 후보 특약과 주석이 실제로 연결되는지 확인한다.
- 이미 명확히 연결된 항목은 자동 반영 가능으로 본다.
- 여러 특약에 공통으로 걸린 주석은 자동 반영하지 않고 확인 질문으로 넘긴다.

### 3차 반영 규칙

- 즉시 반영 가능:
  - 대상 특약 1개만 변경
  - 주석 1개가 그 특약에만 연결
  - 수치 변경이 명확
- 확인 질문 필요:
  - 여러 특약에 영향을 줄 가능성
  - 본문 주석과 비고의 연결이 애매함
  - 변경 범위가 상향인지 예외 수정인지 불분명함

## 주석 병합 규칙

`Note Master`는 두 소스를 합친다.

1. 표 내부 `비고`
   - `주1`, `주2,3` 같은 라벨을 읽고
   - 표 아래 본문 주석 텍스트를 역참조해서 `주석 원문`에 실제 설명문을 넣는다.

2. 표 아래 본문 주석
   - `주1) ...`, `주2) ...` 형태를 별도 행으로 저장한다.
   - `noteType`은 `본문주석`, `계산식`, `예외`로 분류한다.

3. 병합 결과
   - `주석명`은 짧은 참조가 아니라 설명 가능한 이름으로 만든다.
   - `주석 원문`에는 실제 문장과 계산식이 보이게 한다.
   - 계산식이 포함된 항목은 설명문 뒤에 그대로 유지한다.

## 데이터 흐름

1. 업로드 엑셀을 읽어 `Rule Master`, `Note Master`, `Rule-Note Map`, `Change Log`를 만든다.
2. 사용자가 변경내용을 입력하면 영향 특약을 찾는다.
3. 해당 특약의 한도와 연결 주석을 함께 검토한다.
4. 자동 반영 가능한 값은 초안에 즉시 반영한다.
5. 불확실한 연결은 확인 질문 카드에 추가한다.
6. 최종적으로 통합 마스터와 상품 추출 파일을 만든다.

## 사용자 경험

- 사용자는 “변경내용”만 넣어도 된다.
- 시스템은 내부적으로 한도 변경과 주석 변경을 분리해서 판단한다.
- 자동 반영이 가능한 경우에는 질문 수를 줄인다.
- 애매한 경우에는 무리해서 반영하지 않고 질문을 보여준다.

## 에러 처리

- 주석 라벨을 찾지 못하면 해당 주석은 보류하고 확인 질문에 올린다.
- 한도 값이 여러 개로 해석되면 자동 반영하지 않는다.
- 특약명이 중복되면 보험코드 우선으로 처리한다.
- 본문 주석이 누락된 파일은 `검토 필요`로 남긴다.

## 테스트 기준

- `소액암진단 단일건 1000 → 2000` 변경 시 소액암진단만 우선 수정된다.
- 주석이 연결된 특약은 해당 특약에만 따라간다.
- 연결이 불명확한 주석은 질문으로 넘어간다.
- `Note Master`의 `주석 원문`은 라벨만이 아니라 실제 본문 텍스트를 포함한다.
- `비고`와 본문 주석이 함께 있는 파일에서 둘이 한 구조로 보인다.

## 성공 기준

- 전체 특약이 한 번에 잘못 바뀌지 않는다.
- 영향 특약만 정확히 반영된다.
- 주석 원문이 실제 문장으로 보인다.
- 자동 반영과 사람 확인이 명확히 나뉜다.
