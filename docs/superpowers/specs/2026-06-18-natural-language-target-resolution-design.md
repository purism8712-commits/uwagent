# 자연어 변경 대상 해석 설계서

## 목적

사용자가 `"소액암진단 5천으로 변경"` 같은 자연어로 변경 내용을 입력했을 때,  
공통 코어는 이 문장을 바로 전체 특약에 퍼뜨리지 않고 **정확한 특약명/보험코드 기준으로만** 변경 대상을 좁힌다.  
정확히 맞는 대상이 있으면 그 특약만 우선 반영하고, 애매하면 **유사 후보 최대 3개**만 보여 준 뒤 사람 확인을 받는다.

## 배경

현재 공통 코어는 업로드된 가이드라인 엑셀을 읽어 `Rule Master`, `Note Master`, `Rule-Note Map`, `Change Log`를 만드는 역할을 한다.  
다만 자연어 입력이 들어오면 문맥이 비슷한 다른 특약까지 같이 잡히거나, 반대로 매칭이 너무 약해서 잘못된 대상이 선택되는 문제가 있었다.  
특히 같은 상품 안에 비슷한 특약명이 여러 개 있거나, 주석이 여러 특약에 연결된 경우에는 대상 해석이 먼저 정확해야 한다.

## 목표

1. 자연어 입력에서 변경 대상을 먼저 찾는다.
2. 특약명, 보험코드, 상품명은 **정확 매칭을 1순위**로 처리한다.
3. 정확 매칭이 없거나 여러 개면 **유사 후보 최대 3개**만 제시한다.
4. 사용자가 후보를 선택하기 전에는 변경을 초안에 반영하지 않는다.
5. 대상이 확정되면 그 특약에 연결된 한도와 주석만 이어서 검토한다.

## 범위

### 포함

- 자연어 입력에서 특약명/보험코드/상품명 추출
- 정확 매칭 후보 탐색
- 유사 후보 랭킹 및 최대 3개 제시
- 후보 선택 후 단일 특약 반영
- 대상이 불명확할 때 확인 질문 생성

### 제외

- 최종 인수기준 승인
- 유사 특약 최종 결론
- 운영 시스템 반영
- 외부 검색이나 사전학습된 일반 추천

## 설계 원칙

- **정확 매칭 우선**: 보험코드가 있으면 보험코드를 가장 먼저 본다.
- **특약 단위 반영**: 대상이 확정되면 같은 상품의 다른 특약으로 자동 확산하지 않는다.
- **짧고 보수적인 후보 제시**: 유사 후보는 최대 3개까지만 보여 주고, 더 넓게 풀지 않는다.
- **사람 확인 우선**: 불확실하면 자동 반영하지 않고 확인 질문으로 넘긴다.
- **LLM은 재판정용**: 후보를 넓히는 도구가 아니라, 이미 좁혀진 후보를 해석하는 보조로 쓴다.

## 대상 해석 규칙

### 1. 정확 매칭

다음 항목이 자연어 입력에 정확히 포함되면 우선 후보로 잡는다.

- 보험코드
- 특약명
- 상품명

예시:

- `소액암진단 5천으로 변경`
- `LI00113 한도 변경`
- `실버보험 뇌혈관진단 상향`

이 경우 먼저 해당 특약의 현재 기준값과 변경값을 비교한다.  
현재 기준값과 변경값이 같으면 **변경 없음**으로 보고, 바로 반영하지 않고 확인 질문으로 돌린다.

### 2. 유사 후보

정확 매칭이 없으면 다음 기준으로 유사 후보를 찾는다.

- 띄어쓰기 차이
- `주보험 / 주 보험`
- `뇌혈관진단 / 뇌혈관 진단`
- 특약명과 보험코드의 부분 포함

유사 후보는 최대 3개까지만 보여 준다.  
후보가 3개를 넘으면 나머지는 숨기고, 사용자가 선택한 후보만 다음 단계로 넘긴다.

### 3. 대상 불명확

다음 경우에는 자동 반영하지 않는다.

- 입력이 너무 짧아 어떤 특약인지 불명확한 경우
- 같은 문구가 여러 상품/특약에 동시에 걸리는 경우
- 현재 기준값과 변경 요청값이 같은 경우
- 한도 변경과 주석 변경이 함께 섞여 있으나 범위가 분명하지 않은 경우

이때는 확인 질문으로 보낸다.

## UI 흐름

### 1. 변경내용 입력

사용자는 자연어 또는 표 형태로 변경 내용을 입력한다.

### 2. 대상 후보 창

입력 후 바로 아래 또는 별도 창에서 다음을 보여 준다.

- 정확 매칭 결과
- 유사 후보 최대 3개
- 선택된 후보의 상품명, 특약명, 보험코드

### 3. 확정 단계

사용자가 후보를 확정하면 그때만 다음 단계로 넘어간다.

- 검토 메모 생성
- 확인 질문 생성
- Step 4 초안 생성 결과 반영

## 데이터 흐름

1. 사용자가 자연어 변경 내용을 입력한다.
2. 입력에서 상품명, 특약명, 보험코드를 추출한다.
3. 기준 템플릿의 `masterProducts`에서 정확 매칭을 먼저 찾는다.
4. 정확 매칭이 없으면 유사 후보를 최대 3개 계산한다.
5. 후보가 여러 개면 사용자가 하나를 고른다.
6. 선택된 특약만 `Rule Master`와 연결된 `Note Master`를 따라 변경 검토한다.
7. 대상이 확정된 뒤에만 확인 질문과 초안을 생성한다.

## 질문 생성 규칙

- 대상이 확정되기 전의 질문은 **대상 확정 질문**만 허용한다.
- 대상이 확정된 뒤의 질문은 **영향 주석/예외/동일 값 여부** 같은 실제 검토 질문으로 바꾼다.
- 대상이 불명확하면 넓은 범위의 질문을 여러 개 던지지 않는다.
- 사람 확인이 꼭 필요한 항목만 남기고, 나머지는 자동 후보에서 제외한다.

## 에러 처리

- 후보가 0개면 “일치 대상 없음”으로 표시하고 재입력을 유도한다.
- 후보가 여러 개인데 사용자가 선택하지 않으면 다음 단계로 못 간다.
- 보험코드가 들어왔는데 특약명이 다른 경우는 보험코드를 우선한다.
- 현재 기준값과 요청값이 같으면 “변경 없음”으로 남기고, 확인 질문에서 재확인한다.

## 테스트 기준

- `소액암진단 5천으로 변경` 입력 시 소액암진단만 우선 후보가 된다.
- `뇌혈관 진단`과 같이 띄어쓰기가 섞여도 동일 특약이 잡힌다.
- 정확 매칭이 없으면 최대 3개 후보만 보인다.
- 현재 기준값과 요청값이 같으면 변경 반영 대신 확인 질문이 나온다.
- 후보 확정 전에는 Step 4 초안이 생성되지 않는다.

## 성공 기준

- 자연어 입력이 전체 특약으로 퍼지지 않는다.
- 정확 매칭이 있으면 해당 특약만 잡힌다.
- 유사 후보는 제한적으로만 제시된다.
- 사용자가 후보를 확정해야만 초안이 진행된다.
