# 공통 코어 에이전트 앱 설계서

## 목적

이 앱은 사용자가 변경된 엑셀, 표, 자연어 입력을 넣으면 공통 코어 에이전트가 초안을 만들고, 검토메모와 확인 질문을 거쳐 최종 초안 엑셀 생성으로 이어지는 데스크톱용 업무 앱이다.

## 사용자

- 내부 업무 담당자
- 데스크톱 환경에서 신계약 기준 변경 자료를 입력하는 사용자

## 핵심 사용자 흐름

1. 사용자가 첫 화면에서 변경 자료를 입력한다.
2. 입력 방식은 `엑셀 업로드`와 `표/자연어 직접 입력`을 모두 지원한다.
3. 사용자가 `입력완료` 버튼을 누른다.
4. 앱은 2단계 화면으로 이동한다.
5. 2단계 화면에서 검토메모와 확인 질문을 보여준다.
6. 사용자가 질문에 답변한다.
7. 이후 초안 확정 및 엑셀 출력으로 이어질 수 있는 상태를 만든다.

## 정보 구조

### 1단계 입력 화면

- 히어로 섹션 제목: `신계약 공통 에이전트`
- 참고 사이트 분위기를 반영한 금융 서비스형 카드 레이아웃
- `변경내용 입력` 메인 카드
- `엑셀 업로드` 영역
- `표/자연어 직접 입력` 텍스트 영역
- `입력완료` 버튼

### 2단계 검토 화면

- 상단 요약 카드: 업로드 파일명 또는 입력 요약
- 메인 카드: `검토메모`
- 보조 카드: `확인 질문`
- 질문별 답변 입력 필드
- 향후 연결용 CTA: `초안 확정` 또는 `엑셀 초안 생성`

## UI 방향

- 데스크톱 전용 2단계 위저드형 앱
- 참고 사이트의 넓은 여백, 금융 서비스형 정보 배치, 대형 히어로 카드 분위기를 반영
- 시각 언어는 복제하지 않고 재해석
- 메인 컬러는 짙은 네이비, 보조는 소프트 블루와 오프화이트 계열
- 입력/검토 카드는 둥근 모서리와 넉넉한 패딩 사용
- 상태 강조는 블루, 검토 필요는 앰버 계열 사용

## 상태 모델

- `step`: `input` | `review`
- `uploadedFile`: 업로드된 파일 정보
- `rawInput`: 표/자연어 입력 내용
- `reviewMemos`: 검토메모 리스트
- `questions`: 확인 질문 리스트
- `answers`: 질문 응답값

## 초기 구현 범위

- Next.js 기반 앱
- 버튼 클릭 시 실제 2단계 화면 전환
- 파일 업로드 UI
- 텍스트 입력 UI
- 샘플 검토메모/질문 데이터 렌더링
- 질문 응답 입력 UI

## 테스트 기준

- 첫 화면에서 파일 업로드 UI와 텍스트 입력 UI가 모두 보인다
- `입력완료` 클릭 시 2단계 검토 화면으로 이동한다
- 검토메모와 질문 카드가 렌더링된다
- 질문에 답변을 입력할 수 있다
- 데스크톱 해상도에서 레이아웃이 무너지지 않는다

## 구현 메모

- 현재 저장소에는 웹 앱 코드가 없으므로 새 Next.js 앱 구조를 만든다
- `.env`는 이미 준비되어 있으므로 이번 단계에서는 읽기만 고려하고 실제 서버 연동은 하지 않는다
- 공통 코어 에이전트 목적상 화면 문구는 `초안`, `검토 필요`, `확정 전` 표현을 유지한다
