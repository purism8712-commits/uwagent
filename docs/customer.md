# Customer — 공통 코어 에이전트

## ICP

- Primary: 보험 신계약 기준 변경을 여러 후속 에이전트와 사람 검토 흐름에 연결하려는 업무 조직
- Secondary: 기준 원문을 구조화 데이터로 바꾸는 내부 AX/업무혁신 담당자
- Reviewer: 보안, 준법, 운영 리스크를 확인하는 관리자

## 페르소나

| | A. 입력 템플릿 관리자 | B. 후속 에이전트 설계자 | C. 보안·준법/관리자 |
|---|---|---|---|
| 핵심 고통 | 원본 표와 주석을 일관된 구조로 옮기기 어렵다 | 같은 기준을 에이전트마다 다르게 읽는다 | 내부 기준 원문 노출, 자동 판단 확대가 우려된다 |
| 현재 대안 | 수작업 엑셀 정리 | 에이전트별 개별 파싱 | 수동 검토, 사용 제한 |
| 기대 결과 | 입력 템플릿 초안 자동화 | 공통 데이터 구조 확보 | 초안 보조 범위와 안전 경계 명확화 |

## JTBD

- A: 원본 기준표를 규칙, 주석, 변경 이력으로 빠르게 분리하고 싶다
- B: 후속 에이전트가 공통으로 쓸 수 있는 표준 입력 구조가 필요하다
- C: 자동 판단 없이, 안전하게 구조화만 수행되는지 확인하고 싶다

## 성공 경험

한 가지 기준 자료를 넣으면 규칙 본문, 주석, 규칙-주석 연결, 변경 이력이 분리된 입력 템플릿 초안이 생성된다. 담당자는 이 초안을 검토·수정하고, 이후 기획·지원·심사 에이전트가 같은 구조를 사용한다.
