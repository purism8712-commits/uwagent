# PRD — 공통 코어 에이전트

## 문제 정의

보험 신계약 기준은 표, 이미지, 엑셀, 주석, 예외, 계산식이 섞여 있어 사람이 직접 읽을 때도 해석 차이가 생기기 쉽다. 이 상태로 기획, 지원, 심사 에이전트를 바로 연결하면 같은 기준 변경도 서로 다른 구조로 읽어 누락과 재작업이 발생할 수 있다.

## 대상 사용자 / JTBD

- 1차 사용자: 신계약 기획, 지원, 심사 에이전트를 설계하거나 운영하는 담당자
- 2차 사용자: 기준 입력 템플릿을 관리하는 업무 담당자
- JTBD: 원본 기준표를 후속 에이전트가 공통으로 읽을 수 있는 구조화 입력 템플릿으로 빠르고 안전하게 바꾸고 싶다

## 목표

원본 기준 자료를 읽어 `Overview`, `Rule Master`, `Note Master`, `Rule-Note Map`, `Change Log` 구조의 초안 템플릿으로 변환한다. 결과는 사람이 검토 가능한 초안이어야 하며, 원문 출처를 추적할 수 있어야 한다.

## 핵심 기능

### 1. 본문 규칙 구조화

- 입력: 기준표 이미지, 엑셀, 문서 발췌
- 처리: 규칙 본문을 행 단위로 분해하고 as-is/to-be, 조건, 상태를 정리
- 출력: `Rule Master`와 `Overview`의 상품 기본정보

### 2. 주석/예외/계산식 분리

- 입력: 본문 외부 주석, 비고란, 예외 문구, 계산식
- 처리: 주석성 정보를 별도 엔터티로 분리
- 출력: `Note Master`

### 3. 규칙-주석 연결

- 입력: 본문 규칙과 주석 정보
- 처리: 어떤 주석이 어떤 규칙에 어떤 방식으로 적용되는지 연결
- 출력: `Rule-Note Map`

### 4. 변경 이력 정리

- 입력: 변경 전후 한도, 상태 전환, 적용일 메모
- 처리: 현행/종료/시행예정 정보와 변경 이력 분리
- 출력: `Change Log`

## 안 만드는 것

- 인수기준 최종 판단
- 유사 특약 비교 결과 확정
- 시스템 영향도 분석
- RD 반영 항목 확정
- 공지문/FAQ 생성
- 자동 승인, 자동 반영, 자동 발송

## 입력 데이터

- 신계약 가이드라인 표
- 기준표 이미지
- 엑셀 시트
- 주석, 예외, 계산식 포함 문서
- 변경 전후 이력 자료

## 출력 형식

- 엑셀 입력 템플릿
- Markdown 표
- CSV/JSON 구조화 결과

기준 엑셀 템플릿은 `상품명`, `판매일자`를 담는 `Overview` 시트와, 각 상세 시트에 `상품명` 컬럼이 포함된 구조를 기본 계약으로 사용한다.

## 사람 승인 지점

- 구조화 결과 검토
- 주석 연결 검토
- 변경 이력 해석 검토
- 불확실 항목 확인

## 완료 정의

기준 자료 1건에 대해 `Overview`, `Rule Master`, `Note Master`, `Rule-Note Map`, `Change Log`가 생성되고, 불확실 항목은 `검토 필요`로 남아 있으며, 후속 에이전트가 공통 입력으로 사용할 수 있는 상태다.

## 안전 원칙

- 원문에 없는 값은 추가하지 않는다
- 개인정보, 계약번호, 주민번호는 남기지 않는다
- 과도한 원문 복사는 피하고 출처위치만 남긴다
- 모든 결과는 `초안` 상태이며 업무 확정 데이터가 아니다
